Hi hi people, this is Nina! I'm the coder of the 2021DE mod and creator of this site. I’m starting a little blog in this space, for a few reasons: to have a central place for all my guides and coding tips, to share some insights into the modding process that might be useful to other modders and interesting to everyone else, and last but not least to show that this community is still going strong after unfortunate recent incidents. For the first blog, I want to rehash a coding feature that I’ve actually written a guide on reddit about before, but that I’ve gotten asked about a few times recently since many people seem to have been unaware of the reddit post - Tooltips, as first seen in 2021DE:

Let’s start with some code: https://pastebin.com/raw/3WZEeKDH
Put this in your code 1. You can also put it in your code 2 if you want - as long as you put it above the other code, everything should be fine. To add tooltips, simply add new searchStrings in the same format. As you can see with the image example, you can use HTML tags, so add pictures, make them bold, colourful, whatever. To change the background colour of the tooltip area, simply change the background-color of .mytooltip .mytooltiptext ... to something other than lightblue (either a named colour or a hex value - https://htmlcolorcodes.com/ is good for picking colours). In a similar vein, changing the color attribute in the same place changes the color of the text.

The default tooltip look

Something a bit darker...

...or more like a danger sign is also possible

Now the second bit of code: https://pastebin.com/raw/c2DMtD34
Put that part in your code 2, below the part where you define the questions, answers and feedback. It will apply a tooltip to every searchString you defined in the other file to all questions, answers and feedbacks (not to other stuff like endings and depending on how you do your CYOA, not necessarily to those paths either). Cases like overlapping tooltips, fanciful punctuation within the searchStrings and such are mostly dealt with, though there are probably some weird edge cases.

Note, that this will not apply tooltips to your code 1 and endings, I might cover that in a later blogpost. Some technical explanation for the people who are interested - what this code does is basically define pairs of words to explain and their explainer text, then search through all questions, answers and feedback and add some HTML with the explainer tags within specific tags directly behind the word to be explained. Then, CSS is used to hide the tooltips by default and only show them in a box of the right size on hover. You can adjust the looks of it too - the advisor speech bubbles in 2021DE use basically the same system, with some slight styling modifications.

With a little CSS magic, you can turn the tooltips into these advice boxes

Background images also work - to make them look like a letter

Now, what are tooltips good for? In 2021, we mainly used them for three things - giving more background on certain people, events and organizations, showing iconic pictures (like the Laschet laugh after the flood) and for memes (though we used that sparingly). W uses tooltips mainly to explain the nicknames that are used, and sometimes almost like a neutral observer commenting in a less biased way. Some othher uses I came up with are:

  • Highlighting certain events in alternate history mods to give context without overloading the question
  • Fact-checking things the candidate says in answers
  • Give context to offensive or discriminatory statements
  • For foreign mods, give context to words or concepts that don't translate well
  • Use them as advice boxes - maybe your VP can chime in what they think of certain answers?
I'm sure you can come up with even more ways to use them - in any case, they are quite easy to set up, so if you have an idea how to use them, go ahead and try it out!

It's not overstatement that this image decided the 2021 German election - and tooltips were a great way to organically put it into the mod.
