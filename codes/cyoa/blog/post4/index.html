What is an Object in Javascript?

First, let’s take a step back - what even is an object? Not in a philosophical sense, but in a Javascript programming sense. Broadly speaking, an object is a collection of things, where each thing has a key (the name of the thing) and a value (basically what the thing is). An Object can also be nested and contain other objects. A simple object could look like this:

const president = {
  firstName: "Joe",
  lastName: "Biden",
  party: "Democrats",
  vicePresident: {
    firstName: "Kamala",
    lastName: "Harris"
  }
};

(By the way, remember the liberalMap from last week? That was also an object).

The cool thing about objects is that it’s quite simple to access its properties if you know the keys. To access the name of the president in the above object, you simply need to use president.firstName That would have the value “Joe”. This also works with the nested objects inside, so president.vicePresident.firstName would give you “Kamala”. With the same syntax, you can also change the value - for example if you execute this code:

president.vicePresident.firstName = “Gretchen”;
president.vicePresident.lastName = “Whitmer”;

Your object would then look like this:

{   firstName: "Joe",
  lastName: "Biden",
  party: "Democrats",
  vicePresident: {
    firstName: "Gretchen",
    lastName: "Whitmer"
  }
}

Arrays in Javascript

Another thing that exists in Javascript (and can be contained within objects) are arrays. In contrast to objects with their key-value-pairs, arrays are basically just an (ordered) list of things. Let’s add one to our object:

president.electionYears = [2020, 2024] //we can just add a new key-value-pair to our object like this

Now, the president object looks like this:

{   firstName: "Joe",
  lastName: "Biden",
  party: "Democrats",
  vicePresident: {
    firstName: "Gretchen",
    lastName: "Whitmer"
  },
electionYears: [2020, 2024]
}

To access a value in an array, we can’t use a key, since there is none! Instead, we have to work with the index (or position). Important here, computers start counting at zero! Therefore president.electionYears[1] returns us the second element in the electionYears array in the president object - so 2024.

Understanding campaignTrail_temp

Now, you probably already guessed it - campaignTrail_temp is an object. It is not just any object though, it is quite a large object, deeply nested in part, that contains basically everything information of the current scenario! All the candidates, questions, effects and so are all part of this object! In fact, both code 1 and code 2 of a Campaign Trail mod are mostly just assigning values to sub-objects of campaignTrail_temp (or e. e and campaignTrail_temp refers to the same thing and can be used interchangeably). What’s not part of the campaignTrail_temp object is custom Javascript code, like a music player, CYOA, or the modern way of doing endings.

If you want to see how the object looks like, I’d recommend to look at it in the browser console. To do so, start a mod, then right-click anywhere, select “Inspect element”, navigate to the console tab in the thingie that opens up and send the following command on the console:

console.log(campaignTrail_temp)

That will print the whole object to the console - you can click the little arrows to look at specific parts.

Why does this matter?

So, what does this mean? Why do we care that all the game info is stored in a specific Javascript structure? Because it is now clear how to look at and manipulate all the game data. Do you want to read out the difficulty, perhaps because a certain ending can only be achieved on normal? Use campaignTrail_temp.difficulty_level_multiplier. Want to access the colour of the third candidate you defined in your code one? Use campaignTrail_temp.candidate_json[2].fields.color_hex. This works with basically everything, every value of the scenario that’s defined inside the campaignTrail_temp object can be manipulated like this.

So, is this blog already coming to an end then? All you need to do is apply this knowledge? No! Doing this all manually can be very tedious, especially, when working with arrays, since you often don’t really know the index of the thing you want to manipulate. To make things simpler, you often should use functions that do specific things, like exchanging and opposing candidates, or changing the effect of a certain answer.
